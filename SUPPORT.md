# Support

## Public Repositories

For public repositories,
use GitHub issues when issue tracking is enabled on that repository.

If a repository does not use issues,
use the project links published in the repository
or on [staroslabs.dev](https://staroslabs.dev).

## Organization Profile

Questions about the organization profile
or shared community-health files can be reported
in the [`Staros-Labs/.github`](https://github.com/Staros-Labs/.github)
repository.

## Security Issues

Do not use public issues for security reports.
Follow [`SECURITY.md`](SECURITY.md) instead.
